## Overview:
Completed a hands-on lab on Amazon Web Services (AWS) focused on launching and managing EC2 instances, both manually and programmatically, while building a secure and scalable architecture.

## Key Skills Demonstrated
- EC2 Instance Management: Launched and configured EC2 instances using the AWS Management Console.
- Secure Access: Used EC2 Instance Connect to securely access instances, including a bastion host setup.
- Automation: Programmatically launched and configured EC2 instances using the AWS CLI, including setting up a web server with automated user data scripts.
- Networking & Security: Configured VPC, subnets, and security groups to ensure proper access and firewall rules.
- Infrastructure as Code Principles: Learned the benefits of repeatable deployments using CLI commands and AWS CloudFormation for multi-resource launches.
- Troubleshooting: Identified and resolved misconfigurations in instances and web server setups.

## Lab Architecture
- Bastion Host: EC2 instance used as a secure gateway for accessing other instances.
- Web Server Instance: EC2 instance configured via CLI and user data script to host a web application.
- VPC & Security Groups: Ensured network isolation and secure inbound/outbound rules for SSH and HTTP traffic.
- Tools & Services Used
- AWS Management Console – Manual instance launch and configuration
- AWS CLI – Automated instance provisioning and configuration
- EC2 Instance Connect – Secure access to instances
- Amazon Linux 2 AMI – EC2 operating system
- Amazon VPC, Subnets & Security Groups – Networking and security
- User Data Scripts – Automated web server setup
- AWS Systems Manager Parameter Store – Dynamically retrieved latest AMI IDs
